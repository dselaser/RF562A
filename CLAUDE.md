## PPT 작성 규칙 (*_설명.pptx)

### 줄간격
- 본문 텍스트: 행간 **1.5배** (`<a:lnSpc><a:spcPct val="150000"/>`)
- **테이블 셀 내부: 줄간격 적용 금지** (기본값 유지, `<a:tbl>` 안에 lnSpc 넣지 말 것)
- 번호 리스트(`<a:buChar>`) 단락에도 동일하게 1.5배 적용

### 세로 배치
- 제목+구분선은 고정 (이동하지 않음)
- 콘텐츠(테이블, 텍스트)는 **위 여백:아래 여백 = 3:7** 비율로 배치
- 계산: `target_above = (슬라이드높이 - 제목하단 - 콘텐츠높이) * 0.31`
- 콘텐츠 간 상대 위치 유지하면서 그룹 이동

### 테이블 배치
- 테이블을 제목 아래 가용공간에서 3:7 비율 위치에 배치
- 설명 텍스트는 테이블에서 한줄 띄어 배치
- 테이블 여러 개: 중앙에 배치하고, 테이블 간 2~3줄 공간

### 첫/끝 슬라이드
- 배경색: 무색 (투명, `<p:bg>` 블록 제거)
- 회사명: **DSE INC**
- 텍스트 색상: 흰 배경에 맞는 어두운 색

### 플로우차트
- **B/W만** 사용 (흰 배경 + 검정 테두리 + 검정 글자)
- 박스 크기: 글자 양쪽 2글자 공간만 (lIns/rIns = 72000)
- 연결선: 가는 선 (w="9525") + 삼각형 화살촉
- 분기(다이아몬드): Yes/No 라벨, 수직 하강선으로 연결
- 내부 구현 용어(매직값 등) 사용 금지, 쉬운 표현 사용


## 작업 워크플로우
1. 코드 수정 → 빌드 → 기능 시험
2. 사용자가 **"Good"** 이라고 하면 → PPT(_설명.pptx)에 내용 정리
3. PPT 정리 후 → **git commit + push** (코드 + PPT 함께)
4. "Good" 전에는 git에 올리지 않음
